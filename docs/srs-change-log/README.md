# SRS Change Log

This folder tracks implementation decisions that intentionally differ from the current SRS baseline in `Software Requirements Specification (SRS).pdf`.

Use this folder during development to record:
- the original SRS expectation
- the implemented or newly agreed behavior
- why the change was made
- which files or modules were affected
- whether the main SRS should be updated later

Suggested entry format:

```md
# Change 001 - Short Title

Date: YYYY-MM-DD
Status: proposed | accepted | implemented

SRS baseline:
- ...

New decision:
- ...

Reason:
- ...

Affected areas:
- ...

Follow-up for final SRS update:
- ...
```
